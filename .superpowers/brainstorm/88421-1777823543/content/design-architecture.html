Gesamt-Architektur & Datenfluss

Neue Contracts

Neu

character.contract.smart.c

Thin Proxy. Speichert Zustand, leitet Angriffe weiter, verarbeitet eingehende Messages vom Construct.

Erweitert

construct.contract.smart.c

Erkennt Character-Angriffe via getCodeHashOf, wendet Attribute an, führt Counter-Attacks aus, sendet Drops/Buffs.

Angriffs-Fluss

Spieler-Wallet ──attack TX (SIGNA + opt. 1 Token)──▶ Character Contract
Character Contract ──forward (SIGNA + attr-msg + opt. Token)──▶ Construct
Construct ──validates getCodeHashOf(sender)──
Construct ──XP Tokens + outcome message──▶ Character
Construct ──counter-attack damage + effects──▶ Character
Construct ──item drop Token (random)──▶ Character
Character ──updates KKV maps (HP, debuffs, items)──
