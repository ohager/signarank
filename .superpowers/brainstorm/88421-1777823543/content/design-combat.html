Design: Abschnitt 2/4 — Attribute & Kampfmechanik

Wie Attribute den Kampf beeinflussen, plus Counter-Attack & Leveling

Attribute-Effekte

STÄRKE (Strength)

+2% Schaden pro Punkt

Schwelle 5: "Berserk" — Crit-Chance +15%

AUSDAUER (Stamina)

+20 max HP pro Punkt

Schwelle 5: "Fortified" — Damage-Received –20%

GESCHICKLICHKEIT (Dexterity)

–1 Block Cooldown pro Punkt

Schwelle 5: "Evasion" — Counter-Attack-Chance –30%

GLÜCK (Luck)

+2% Item-Drop-Chance pro Punkt

Schwelle 5: "Fortune" — Goldener Crit möglich (3× Schaden)

WILLENSKRAFT (Willpower)

–10% Debuff-Dauer pro Punkt / +5% Token-Effekt-Stärke pro Punkt

Schwelle 5: "Iron Will" — Immune gegen Freeze/Stun einmal pro Kampf

Konkrete Zahlenwerte sind konfigurierbar — das sind Vorschläge zum Diskutieren.

Angriffs-Message-Format (Character → Construct)

message[0] = STR | (STA << 8) | (DEX << 16) | (LCK << 24)
message[1] = WIL | (activeItemEffects << 8)
message[2] = characterLevel
message[3] = 0 (reserved)

Attribute werden bitgepacked in einen 64-bit Long — alle 5 passen in 2 message-slots.

Counter-Attack (Construct → Character)

message[0] = COUNTER_ATTACK (magic code)
message[1] = damageAmount
message[2] = statusEffect (0=none, 1=frozen, 2=stunned, 3=weakened)
message[3] = statusDuration (in blocks)

Character reduziert HP-KKV, schreibt Status-Effekt-KKV. Bei HP ≤ 0: Tod-Sequenz.

Leveling

XP = getAssetBalance(this, xpTokenId)

Level-Tabelle: konfigurierbar via Admin-Funktion in Character-Contract
Default: 1000, 2000, 4000, 8000, 16000 ... (×2 pro Stufe)

LevelUp → CHAR_SKILL_POINTS KKV += 1
Spieler verteilt Punkte via Owner-TX mit Attribut-Index in Message
