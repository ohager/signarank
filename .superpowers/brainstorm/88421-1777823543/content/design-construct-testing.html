Testing-Strategie (Prio: Contracts vor Frontend)

1. Unit Tests (Testbed-Modus)
Für jede neue Funktion: applyCharacterModifiers, tryItemDrop, executeCharacterCounterAttack, ALLOCATE_SKILL, Tod-Sequenz, Revival
2. Integration Tests (Character ↔ Construct)
Angriff von Character wird erkannt, Attribute korrekt gelesen, Counter-Attack kommt an, Item-Drop landet im Character
3. Migrations-Test
Alter Character deployen, Attribute setzen, MIGRATE auf neuem Contract aufrufen, Werte vergleichen
