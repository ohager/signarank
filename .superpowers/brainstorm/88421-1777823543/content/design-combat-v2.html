Design: Abschnitt 2/4 — Attribute & Kampfmechanik (v2)

Construct liest Attribute direkt aus Character-KKV — kein Message-Format-Änderung

Angriffs-Flow (unverändert wie heute)

Spieler-Wallet ──TX (SIGNA + opt. 1 Token)──▶ Character Contract
Character ──forward (SIGNA + opt. Token, gleiche Logik)──▶ Construct
Construct ──getCodeHashOf(sender) == CHAR_HASH?──
ja → Construct liest Attribute aus Character-KKV
nein → normale Angriffs-Logik (wie heute)

KKV-Schlüssel (geteilte Konstanten)

Beide Contracts nutzen dieselben Map-Schlüssel — Construct kann direkt lesen:

#define MAP_CHAR_STRENGTH 100
#define MAP_CHAR_STAMINA 101
#define MAP_CHAR_DEXTERITY 102
#define MAP_CHAR_LUCK 103
#define MAP_CHAR_WILLPOWER 104
// Construct liest: getMapValue(senderAccountId, MAP_CHAR_STRENGTH)

Schadensberechnung mit Attributen

// Bestehende Logik
long damage = applyTokenModifiers(calculateSignaDamage());

// Neu: Character-Attribut-Modifier
if (getCodeHashOf(sender) == KNOWN_CHAR_HASH) {
long str = getMapValue(sender, MAP_CHAR_STRENGTH);
long dex = getMapValue(sender, MAP_CHAR_DEXTERITY);
long lck = getMapValue(sender, MAP_CHAR_LUCK);
damage = applyCharacterModifiers(damage, str, dex, lck);
}

Attribut-Effekte

STÄRKE

+X% Schaden pro Punkt
Schwelle 5 → Crit-Chance

AUSDAUER

+X max HP pro Punkt
Schwelle 5 → Damage Resist

GESCHICKLICHKEIT

–X Blöcke Cooldown pro Punkt
Schwelle 5 → Counter-Chance –30%

GLÜCK

+X% Item-Drop-Chance pro Punkt
Schwelle 5 → Goldener Crit

WILLENSKRAFT

–X% Debuff-Dauer, +X% Token-Effekt pro Punkt
Schwelle 5 → Iron Will (einmalige Freeze/Stun-Immunität)
